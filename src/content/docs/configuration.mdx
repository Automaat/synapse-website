---
title: Configuration
description: Full reference for ~/.sybra/config.yaml.
---

Sybra reads `~/.sybra/config.yaml` on startup. Changes require a restart.

## Full example

```yaml
# ~/.sybra/config.yaml

# Which project types this machine handles (empty = all)
project_types: [pet, oss]

github:
  enabled: true
  token: ghp_xxxxxxxxxxxxxxxxxxxx   # Personal Access Token (repo scope)
  poll_interval: 5m

todoist:
  enabled: false
  api_token: xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
  project_id: "12345678"            # Todoist project to pull from
  poll_interval: 5m

renovate:
  enabled: true
  poll_interval: 15m

# Default agent settings
defaults:
  agent_mode: headless
  allowed_tools: []                 # empty = all tools
```

## Fields

### `project_types`

```yaml
project_types: [work]
```

Limits which projects this machine's automations act on. Match against the `type` field set when creating a project. Empty means no filter — all projects are eligible.

### `github`

| Field | Description |
|-------|-------------|
| `enabled` | Enable GitHub Issues import |
| `token` | PAT with `repo` scope |
| `poll_interval` | How often to check for new issues (default `5m`) |

Issues with the label `sybra` (configurable) are imported as tasks. The `project_id` on the resulting task is inferred from the repository.

### `todoist`

| Field | Description |
|-------|-------------|
| `enabled` | Enable Todoist import |
| `api_token` | Todoist API token |
| `project_id` | Todoist project ID to watch |
| `poll_interval` | How often to poll (default `5m`) |

### `renovate`

| Field | Description |
|-------|-------------|
| `enabled` | Enable automatic Renovate PR fixing |
| `poll_interval` | How often to check for open Renovate PRs |

When enabled, Sybra creates headless tasks to fix failing Renovate dependency update PRs.

### `defaults`

Default values applied to all newly created tasks when the field is not explicitly set.

## Environment variables

| Variable | Config field |
|----------|-------------|
| `SYBRA_GITHUB_TOKEN` | `github.token` |
| `SYBRA_TODOIST_TOKEN` | `todoist.api_token` |

Environment variables take precedence over `config.yaml`.

---
title: Task Format
description: YAML frontmatter schema and markdown body conventions for Sybra tasks.
---

Tasks are plain markdown files stored in `~/.sybra/tasks/`. The filename is `<id>.md`.

## Schema

```yaml
---
id: task-abc123            # auto-generated on creation
title: Implement auth middleware
status: todo               # see Status values below
agent_mode: headless       # headless | interactive
allowed_tools: []          # empty = all tools (dangerously-skip-permissions)
tags: [backend, auth]
project_id: owner/repo     # optional — links to a registered project
created_at: 2026-04-02T10:00:00Z
updated_at: 2026-04-02T10:00:00Z
---

## Description

Full task description in GFM markdown. Be specific — this is the prompt
the agent receives. Include context, constraints, and expected output.

## Acceptance criteria

- [ ] JWT validation works with RS256
- [ ] Refresh token rotation implemented
- [ ] Unit tests cover expiry edge cases
```

## Status values

| Status | Meaning |
|--------|---------|
| `new` | Just created, not yet triaged |
| `todo` | Ready to be picked up |
| `in-progress` | Agent is running |
| `in-review` | Agent finished, awaiting human review |
| `human-required` | Agent blocked, needs human input |
| `done` | Completed and merged |

## `allowed_tools`

Controls which Claude Code tools the agent can use.

```yaml
allowed_tools: []                          # all tools (--dangerously-skip-permissions)
allowed_tools: [Read, Edit, Bash]          # specific tools only
allowed_tools: [Read, Grep, Glob]          # read-only mode
```

:::caution
Empty `allowed_tools` passes `--dangerously-skip-permissions` to `claude -p`. Only use for trusted tasks in isolated environments.
:::

## `project_id`

Linking a task to a project enables git worktree isolation:

```yaml
project_id: myorg/myrepo
```

The agent will check out the project's bare clone into a fresh worktree at `~/.sybra/worktrees/<task-id>/` before starting, and clean up on completion.

## Creating tasks

```sh
# Via CLI
sybra-cli create --title "Add rate limiting" --mode headless --tags backend

# Via GUI
# Click "New Task" in the sidebar
```

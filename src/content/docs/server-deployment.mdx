---
title: Server Deployment
description: Run Sybra headless on a home server or VPS with Docker Compose.
---

Sybra runs headless (no GUI) as a Docker container, ideal for a home server or VPS that processes tasks 24/7.

## Docker Compose

```yaml
# /opt/sybra/docker-compose.yml
services:
  sybra:
    image: ghcr.io/automaat/sybra:latest
    restart: unless-stopped
    volumes:
      - /data/sybra/home:/root/.sybra     # tasks, config, clones
      - /data/sybra/claude:/root/.claude    # Claude Code settings + hooks
    ports:
      - "127.0.0.1:8080:8080"                # bind locally, expose via reverse proxy
    environment:
      - SYBRA_GITHUB_TOKEN=${GITHUB_TOKEN}
```

```sh
docker compose up -d
```

## Traefik + Cloudflare TLS

```yaml
# Add to the sybra service labels:
labels:
  - "traefik.enable=true"
  - "traefik.http.routers.sybra.rule=Host(`sybra.example.com`)"
  - "traefik.http.routers.sybra.tls.certresolver=cloudflare"
  - "traefik.http.routers.sybra.middlewares=lan-only@file"
```

Restrict access to LAN + VPN CIDRs via a Traefik middleware file:

```yaml
# traefik/middlewares.yaml
http:
  middlewares:
    lan-only:
      ipAllowList:
        sourceRange:
          - "192.168.0.0/16"
          - "100.64.0.0/10"   # Tailscale
```

## Updating

Bump the image tag in `docker-compose.yml` and redeploy:

```sh
docker compose pull && docker compose up -d
```

Or use the Ansible playbook if you have one configured.

## Multi-machine config

On the server, scope it to the project types it should handle:

```yaml
# /data/sybra/home/config.yaml
project_types: [pet, oss]

github:
  enabled: true
  token: ghp_xxxx

renovate:
  enabled: true

todoist:
  enabled: true
  api_token: xxxx
```

On your laptop, set the complementary types:

```yaml
project_types: [work]

github:
  enabled: true
  token: ghp_xxxx

todoist:
  enabled: false
```

This ensures GitHub Issues for `pet` projects are only picked up by the server, never duplicated on the laptop.

## Claude Code hooks

Mount `/root/.claude` to persist Claude Code configuration, hooks, and the `klaudiush` event monitoring setup:

```
/data/sybra/claude/
├── settings.json        # tool permissions, hooks
└── hooks/               # custom hook scripts
```
